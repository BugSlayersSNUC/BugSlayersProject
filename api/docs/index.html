BugSlayers REST API

A Firebase-authenticated Express API backed by SQLite via Sequelize. Manage users, groups, blood donations, community posts, and comments.

Express.js Sequelize ORM SQLite Firebase Admin JWT Auth
Authentication Required — All endpoints (except POST /api/auth/token) require a JWT in the Authorization header:
Authorization: Bearer <idToken>
🔐 Authentication
Exchange a Firebase ID token for a server-issued JWT used in all subsequent requests.
POST /api/auth/token Exchange Firebase token

Verifies a Firebase ID token and returns a server-issued JWT (idToken) to use in all protected endpoints.

Request Body
FieldTypeRequiredDescription
firebaseTokenstringrequiredFirebase ID token from client
Response
// 200 OK
{ "idToken": "eyJhbGci..." }
GET /api/auth 🔒 Auth Verify token

Returns the authenticated user's Firebase UID. Useful for token verification.

Response
{ "message": "Authenticated successfully", "uid": "abc123" }
👤 Users
Manage user profiles including name, group membership, and donation history.
GET /api/users 🔒 Auth Current user + donations

Returns the full profile of the currently authenticated user along with all their donation records.

Response
{
  "user_id": 1,
  "uid": "firebase_uid_abc",
  "email": "user@example.com",
  "first_name": "Alice",
  "last_name": "Smith",
  "group_id": 2,
  "points": 150,
  "Donations": [
    { "donation_id": 1, "place": "City Hospital", "time": "2026-03-25T...", "points": 50 }
  ]
}
PATCH /api/users 🔒 Auth Update name / group

Update the current user's first name, last name, and/or group. All fields are optional.

Request Body
FieldTypeRequiredDescription
first_namestringoptionalFirst name
last_namestringoptionalLast name
group_idintegeroptionalID of group to join
Response
// Updated user object
{ "user_id": 1, "first_name": "Alice", "group_id": 3, ... }
GET /api/users/:id 🔒 Auth User by ID + donations

Returns the profile and all donations for any user by their integer user_id.

Path Parameters
ParamTypeDescription
idintegeruser_id from the Users table
👥 Groups
Manage blood donation groups. The GET endpoint queries the GroupTotal SQL view for aggregated stats.
GET /api/groups 🔒 Auth All groups with totals

Returns all groups with member count and aggregate points sourced from the GroupTotal SQL view.

Response
[
  {
    "group_id": 1,
    "group_name": "Team Alpha",
    "member_count": 5,
    "total_points": 740
  }
]
POST /api/groups 🔒 Auth Create a group

Creates a new group.

Request Body
FieldTypeRequiredDescription
group_namestringrequiredName of the group
Response
// 201 Created
{ "group_id": 3, "group_name": "Team Gamma" }
💉 Donations
Record and retrieve blood donation events. Points are automatically added to the user's balance on creation.
GET /api/donations 🔒 Auth List donations

Returns donations for the current user by default. Pass ?userid= to get donations for a specific user.

Query Parameters
ParamTypeRequiredDescription
useridintegeroptionalFilter by a specific user_id
Response
[
  {
    "donation_id": 1,
    "user_id": 1,
    "place": "City Hospital",
    "time": "2026-03-25T16:00:00.000Z",
    "points": 50
  }
]
POST /api/donations 🔒 Auth Record donation

Records a donation for the current user. Automatically increments the user's total points by the given amount. Timestamp is set server-side.

Request Body
FieldTypeRequiredDescription
placestringrequiredLocation where donation took place
pointsintegerrequiredPoints to award for this donation
Response
// 201 Created
{ "donation_id": 2, "place": "Red Cross", "time": "...", "points": 50 }
🌐 Community
Community posts with nested comments. Useful for group discussions and announcements.
GET /api/community 🔒 Auth All posts with comments

Returns all community posts ordered by date descending, each with their nested comments. Backed by the CommunityWithComments SQL view.

Response
[
  {
    "community_id": 1,
    "title": "Blood Drive This Weekend!",
    "description": "Join us at...",
    "user_id": 1,
    "upvotes": 12,
    "date": "2026-03-25T...",
    "Comments": [
      { "comment_id": 1, "text": "I'll be there!", "date": "..." }
    ]
  }
]
POST /api/community 🔒 Auth Create post

Creates a new community post authored by the current user. upvotes starts at 0 and date is set server-side.

Request Body
FieldTypeRequiredDescription
titlestringrequiredPost title
descriptionstringrequiredPost body content

📊 SQL Views
Two SQL views are created at startup to improve query performance.

GroupTotal

Pre-aggregates member count and total points per group by joining Groups ← Users. Used by GET /api/groups.

CommunityWithComments

Pre-joins Communities with Comments for efficient reads. Used as a reference layer for GET /api/community.
